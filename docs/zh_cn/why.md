# 为什么不建议拼命打CSP/OI？

> 注：个人观点，仅供参考。本文章及其他文章仅为建议，对比赛持中立态度。

有梦想、有目标是好事，如果你觉得打这些比赛会让你的编程能力有所提升，或者你是为了去锻炼思维，这样对你有好处的话，那就去参加吧，你已经比90%的人都要棒了(๑•̀ㅂ•́)و✧

### 1. 目标不对

你如果只是想学习编程（只想学C++），那么我**不建议**你走这条路。大部分学生学习C++并且参加这个比赛，大概率是因为家长要求的，这是因为他们看中了它的一个特点：**辅助升学**。

不过，随着比赛的低龄化，参加的人数越来越多，这会导致从这个比赛里面取得大成就并且能辅助升学的难度大大提高。***（此处以CSP/J数据作为参考）***

### 2. 适宜人群问题

任何比赛*（不仅仅局限于此）*都是有适宜人群的。一些家长“为了”孩子的未来，迫使他们去学习并且参加这些比赛，这可能会让他们的压力增加（况且这个年龄段基本都在学习），并且可能会付出无效努力，最终取得不到很好的比赛成果。

### 3. C++标准

教打这些比赛的培训机构一般会使用**不标准**的C++语法，虽然这对coding速度有很大提升，不过如果你以后要从事正式的C++开发，这样的OIer-Style语法是很难改的

```cpp
#include<bits/stdc++.h>
using namespace std;
int main()
{
    return 0;
}
```
以上就是一段示例的OIer-Style代码，其使用了“万能头”（`bits/stdc++.h`）及其他的不规范编码。

### 4. 高投入低回报

抛开自学的话题不谈，像这类培训机构一般价格比较昂贵，但是你投入学习之后，不一定能取得什么大成就。

> 作者的话：如果你把投入到CSP/OI的精力放到学习上，你就不需要它们给你带来的升学辅助了。
